# Contributing to AI Threat Intelligence Dashboard

Thank you for your interest in contributing to the AI Threat Intelligence Dashboard! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/ai-threat-intel-dashboard.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Install dependencies: `npm install`
5. Start development server: `npm run dev`

## Development Guidelines

### Code Style

- Follow TypeScript best practices
- Use functional components with hooks
- Maintain consistent naming conventions
- Add comments for complex logic
- Keep components modular and reusable

### Commit Messages

Use clear, descriptive commit messages:

```
feat: Add IP reputation lookup feature
fix: Resolve map rendering issue on mobile
docs: Update README with installation steps
style: Format code with Prettier
refactor: Simplify threat feed component
test: Add unit tests for authentication
```

### Pull Request Process

1. Ensure your code follows the project's style guidelines
2. Update documentation if needed
3. Test your changes thoroughly
4. Create a pull request with a clear description
5. Reference any related issues

## Project Structure

```
src/
├── components/     # Reusable UI components
├── context/       # React context providers
├── pages/         # Page components
├── types/         # TypeScript type definitions
└── utils/         # Utility functions and mock data
```

## Areas for Contribution
